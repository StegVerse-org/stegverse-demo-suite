# Report Formatting Guide

This document defines the standard format for StegVerse runtime demonstration reports.

## Report Naming

StegVerse_<Capability>_Runtime_Transcript_v1.pdf

Examples:

- StegVerse_Governed_Execution_Runtime_Transcript_v1.pdf
- StegVerse_Mutation_Governance_Runtime_Transcript_v1.pdf
- StegVerse_Capability_Release_Runtime_Transcript_v1.pdf

Reports are stored in this directory.

## Report Structure

1. Title Page
2. Executive Summary
3. Command Sequence
4. Runtime Overview
5. Execution Transcript
6. Verification Result
7. Appendix (optional)

Reports should ideally be 6–10 pages.

## Screenshot Rules

Screenshots must include:

- full terminal width
- visible commands
- runtime output
- receipt identifiers when present

Do not crop command prompts.
